📋 Non-Technical Setup Guide

Get Orion live in under 30 minutes

No coding experience needed. Follow each step in order and you'll have your own AI document platform running — for free — with a live URL you can share with anyone.

⏱ ~25 minutes total time
💸 Free on all platforms
🖥 Works on Mac, Windows, Linux
📁
GitHub
Stores your code online so Vercel can read it
🔥
Firebase
Handles login, database & file storage
🚀
Vercel
Hosts your app and gives you a live URL
1
Put your code on GitHub
~5 minutes · GitHub is like a Google Drive for code
1
Create a free GitHub account

If you don't have one, go to github.com and sign up — it's free.

→ Sign up at github.com
2
Create a new repository

A repository (repo) is a folder for your project on GitHub.

Click the + button top-right → New repository
Name it orion-saas (or anything you like)
Set it to Private (keeps your code safe)
Click Create repository
3
Upload your project files

On your new repository page, you'll see an option to upload files directly — no coding needed.

Click uploading an existing file on the repo page
Unzip the orion-saas.zip file on your computer first
Drag the entire orion-saas folder contents into the GitHub upload area
Scroll down and click Commit changes
💡
Tip: Upload the contents of the folder (src, public, package.json etc.) — not the folder itself. You should see files like package.json at the root of your repo.
2
Set up Firebase
~10 minutes · Free Google service — handles login, database & file storage
1
Create a Firebase project
→ Open Firebase Console
Sign in with your Google account
Click Add project
Name it orion-saas
Disable Google Analytics (not needed) → Click Create project
Wait ~30 seconds, then click Continue
2
Enable Authentication

This lets users log in to your app.

In the left sidebar, click Authentication
Click Get started
Click Email/Password → toggle it ON → Save
Click Add new provider → Google → toggle ON → add your email as support email → Save
3
Create Firestore Database

This stores conversations, file info, and user data.

In the left sidebar, click Firestore Database
Click Create database
Select Start in production mode → Next
Choose a location close to you (e.g. us-central1 for USA) → Enable
⚠️
Important: After creating, go to the Rules tab and paste in the security rules below. Without this, your database will block all reads/writes.
Firestore Rules — paste into Rules tab
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId} {
      allow read, write: if request.auth != null
        && request.auth.uid == userId;
    }
    match /files/{fileId} {
      allow read, write: if request.auth != null
        && request.auth.uid == resource.data.uid;
      allow create: if request.auth != null
        && request.auth.uid == request.resource.data.uid;
    }
    match /conversations/{convId} {
      allow read, write: if request.auth != null
        && request.auth.uid == resource.data.uid;
      allow create: if request.auth != null
        && request.auth.uid == request.resource.data.uid;
      match /messages/{messageId} {
        allow read, write: if request.auth != null
          && get(/databases/$(database)/documents/conversations/$(convId))
            .data.uid == request.auth.uid;
      }
    }
  }
}

After pasting, click Publish.

4
Enable Storage

This is where uploaded documents are saved.

In the left sidebar, click Storage
Click Get started → Next → Done
Go to the Rules tab and replace the existing rules with:
Storage Rules — paste into Rules tab
rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    match /users/{userId}/files/{allPaths=**} {
      allow read, write: if request.auth != null
        && request.auth.uid == userId
        && request.resource.size <= 25 * 1024 * 1024;
    }
  }
}

Click Publish.

5
Get your Firebase config keys

These are like a password that connects your app to Firebase. Keep them safe.

Click the ⚙️ gear icon (top-left) → Project settings
Scroll down to Your apps section
Click the </> (Web) icon to add a web app
Give it a nickname like orion-web → click Register app
You'll see a code block — copy the values inside firebaseConfig
ℹ️
The config looks like this — you need to copy each value:

const firebaseConfig = { apiKey: "AIzaSy...", ← copy this authDomain: "your.firebaseapp.com", ← copy this projectId: "your-project", ← copy this storageBucket: "your.appspot.com", ← copy this messagingSenderId: "12345", ← copy this appId: "1:12345:web:abc" ← copy this };

Save these in a text file for the next phase — you'll need them all.

3
Deploy on Vercel
~5 minutes · This gives your app a live URL anyone can visit
1
Create a free Vercel account
→ Sign up at vercel.com
💡
Sign up with GitHub — this makes connecting your repo much easier (one click).
2
Import your GitHub repository
On your Vercel dashboard, click Add New… → Project
Click Import next to your orion-saas repo
Don't click Deploy yet — you need to add environment variables first
3
Add your environment variables

On the import page, scroll down to find Environment Variables. Add each one below — name on the left, value on the right.

Variable NameWhere to find the value
VITE_FIREBASE_API_KEY Firebase config → apiKey
VITE_FIREBASE_AUTH_DOMAIN Firebase config → authDomain
VITE_FIREBASE_PROJECT_ID Firebase config → projectId
VITE_FIREBASE_STORAGE_BUCKET Firebase config → storageBucket
VITE_FIREBASE_MESSAGING_SENDER_ID Firebase config → messagingSenderId
VITE_FIREBASE_APP_ID Firebase config → appId
VITE_GEMINI_API_KEY Your Gemini API key from Google AI Studio
⚠️
All 7 variables are required. If any are missing the app won't work. Double-check for typos — copy and paste the values, don't type them.
4
Deploy!
Click Deploy — Vercel will build and deploy your app (takes ~2 minutes)
You'll see a URL like orion-saas-abc123.vercel.app — that's your live app!
Click it to open your app in the browser
4
One last Firebase step
~2 minutes · Tell Firebase your Vercel URL is allowed to use it
1
Add your Vercel URL to Firebase Auth

Firebase needs to know your app's URL is allowed. Without this, Google login won't work.

Go back to Firebase Console → Authentication
Click the Settings tab → Authorized domains
Click Add domain
Paste your Vercel URL (e.g. orion-saas-abc123.vercel.app) → Add
✅
That's it! Your app is now fully live. Visit your Vercel URL, create an account, upload a document, and start chatting with it using AI.

Common problems & fixes

The app loads but login doesn't work
Make sure you added your Vercel URL to Firebase → Authentication → Settings → Authorized domains. Also check that Email/Password and Google are both enabled in the Sign-in methods tab.
Vercel build fails with an error
Check that all 7 environment variables are added in Vercel → Project Settings → Environment Variables. Even one missing variable will cause the build to fail. Also check that package.json is at the root of your GitHub repo, not inside a subfolder.
File upload fails or shows an error
Make sure you've enabled Firebase Storage and published the Storage Rules above. Also confirm the VITE_FIREBASE_STORAGE_BUCKET variable is correctly set (it should end in .appspot.com).
AI chat doesn't respond or shows an error
Check that your VITE_GEMINI_API_KEY is correct. Get a new key at aistudio.google.com/app/apikey if needed. Make sure the Gemini API is enabled for your Google account — it's free up to generous limits.
I made a change but the live site didn't update
Go to your GitHub repo, edit the file (click the pencil icon), make your change, and click Commit changes. Vercel will automatically detect the change and redeploy within 1–2 minutes.
I want to update my environment variables
Go to Vercel → your project → Settings → Environment Variables. Edit the value and save. Then go to Deployments and click Redeploy on the latest deployment.

In Vercel → your project → Settings → Domains. Add any domain you own — Vercel handles the SSL certificate automatically.

Use the Firebase + Stripe extension to charge users for Pro plan access. Search "Stripe Payments" in the Firebase Extensions marketplace.

Monitor usage
Firebase Console → Usage tab shows reads, writes, and storage used. Vercel Analytics (free) shows visitor traffic.

Currently works best with text files. For full PDF text extraction, upgrade to the Gemini Files API — instructions are in the README.
